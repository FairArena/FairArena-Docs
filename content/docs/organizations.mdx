## Organization Structure

Understanding how organizations work:

| Component | Description | Purpose |
|-----------|-------------|---------|
| **Organization** | Top-level entity | Main container for all collaboration |
| **Teams** | Work groups within org | Focused collaboration spaces |
| **Projects** | Work items within teams | Specific deliverables and tasks |
| **Roles** | Permission sets | Control access and capabilities |
| **Members** | People in the org | Users with assigned roles |

## Joining Organizations

### Three Ways to Join

<Accordions>
  <Accordion title="1. Open Join (Instant Access)">
    Some organizations allow anyone to join instantly:
    
    1. Find the organization
    2. Click **Join Organization**
    3. You're in! üéâ
    
    **Requirements:**
    - Organization must have "Join Enabled"
    - You must have a verified account
    - Default role is usually "Member"
  </Accordion>
  
  <Accordion title="2. Invitation Code (Secure Access)">
    Most organizations use invite codes:
    
    1. Receive invite code via email
    2. Click the invitation link
    3. Review organization details
    4. Accept the invitation
    5. Choose role (if options available)
    6. Confirm and join
    
    **About Invite Codes:**
    - Unique and secure
    - Can be role-specific
    - May have expiration dates
    - Single-use only
    - Sent by organization admins
  </Accordion>
  
  <Accordion title="3. Request Access (Approval Required)">
    For closed organizations:
    
    1. Visit organization profile
    2. Click **Request Access**
    3. Write a brief message (optional)
    4. Submit request
    5. Wait for admin approval
    6. Receive notification when approved
    
    **Tips for requests:**
    - Explain why you want to join
    - Mention relevant experience
    - Be professional and concise
    - Follow up if no response in a week
  </Accordion>
</Accordions>

## Creating an Organization

### Planning Your Organization

Before creating, consider:

| Decision | Options | Impact |
|----------|---------|--------|
| **Purpose** | Company, Community, Project | Shapes structure |
| **Visibility** | Public or Private | Who can see it |
| **Size** | Small team or Enterprise | Resource needs |
| **Structure** | Teams and roles | How people work |

### Creation Steps

<Steps>

### Navigate to Creation

Go to **Organizations** ‚Üí **Create Organization**

### Set Basic Information

**Organization Name**
- Clear and memorable
- Represents your organization well
- Example: "Acme Corp", "React Developers"

**Slug**
- URL-friendly identifier
- Auto-generated from name
- Can be customized
- Example: "acme-corp", "react-devs"

### Write Description

Create a compelling description:
- What your organization does
- Who should join
- What to expect
- Main goals and values

**Example:**
```
Acme Corp is a leading software development company specializing in 
cloud solutions. We build scalable applications using modern technologies 
and welcome developers passionate about innovation.
```

### Upload Branding

**Logo**
- Recommended size: 200x200px
- Formats: PNG, JPG, SVG
- Appears on profile and in listings

**Banner**
- Recommended size: 1200x400px
- Creates visual impact
- Appears on organization page

### Configure Settings

**Visibility Settings:**
- ‚òëÔ∏è Public (visible to all)
- ‚òê Private (members only)

**Join Settings:**
- ‚òëÔ∏è Enable open join
- ‚òê Invitation only

**Other Settings:**
- Set timezone
- Add location
- Add website URL

### Create Initial Roles

Set up role structure (you can modify later):

| Role | Permissions | Suggested For |
|------|-------------|---------------|
| **Owner** | Full control | Founders |
| **Admin** | Manage most things | Leadership team |
| **Member** | Basic access | Regular employees |
| **Viewer** | Read-only | Contractors, observers |

### Invite Initial Members

Add your first team members:
- Enter email addresses
- Assign roles
- Send invitations

### Launch!

Click **Create Organization** and you're live! üöÄ

</Steps>

## Managing Your Organization

### Member Management

**Adding Members:**

<Tabs items={['Invite by Email', 'Invite Code', 'Open Join']}>
  <Tab value="Invite by Email">
    1. Go to **Organization Settings**
    2. Click **Members** ‚Üí **Invite**
    3. Enter email addresses (one per line)
    4. Select role for invitees
    5. Add personal message (optional)
    6. Click **Send Invitations**
    
    Recipients receive email with join link.
  </Tab>
  <Tab value="Invite Code">
    1. Go to **Organization Settings**
    2. Click **Invite Codes** ‚Üí **Create**
    3. Select role for code
    4. Set expiration date
    5. Optional: Specify email
    6. Click **Generate Code**
    7. Share code with invitee
    
    Code can be used once before expiring.
  </Tab>
  <Tab value="Open Join">
    1. Go to **Organization Settings**
    2. Enable **Open Join**
    3. Set default role for new members
    4. Save settings
    
    Anyone can now join instantly!
  </Tab>
</Tabs>

**Removing Members:**

1. Go to **Members** list
2. Find the member
3. Click options menu (‚ãØ)
4. Select **Remove from Organization**
5. Confirm removal

<Callout type="warning">
  Removing a member also removes them from all teams and projects within the organization.
</Callout>

## Organization Roles & Permissions

### Standard Roles Explained

<Tabs items={['Owner', 'Admin', 'Member', 'Viewer']}>
  <Tab value="Owner">
    **Highest level of control**
    
    **Can:**
    - ‚úÖ Delete the organization
    - ‚úÖ Transfer ownership
    - ‚úÖ Manage all settings
    - ‚úÖ Add/remove members (including admins)
    - ‚úÖ Assign any role
    - ‚úÖ Manage billing
    - ‚úÖ View audit logs
    - ‚úÖ Everything admins can do
    
    **Limits:**
    - Only one owner per organization
    - Can transfer to another member
    
    **Best for:** Founders, CEO
  </Tab>
  <Tab value="Admin">
    **High level of control**
    
    **Can:**
    - ‚úÖ Manage organization settings
    - ‚úÖ Create and manage teams
    - ‚úÖ Invite members
    - ‚úÖ Assign roles (except owner)
    - ‚úÖ Remove members (except owner)
    - ‚úÖ Configure team settings
    - ‚úÖ View audit logs
    
    **Cannot:**
    - ‚ùå Delete organization
    - ‚ùå Remove owner
    - ‚ùå Assign owner role
    - ‚ùå Transfer ownership
    
    **Best for:** Leadership team, managers
  </Tab>
  <Tab value="Member">
    **Standard access**
    
    **Can:**
    - ‚úÖ View organization information
    - ‚úÖ Join teams (if allowed)
    - ‚úÖ Participate in projects
    - ‚úÖ View other members
    - ‚úÖ Create teams (if allowed)
    - ‚úÖ Follow and star
    
    **Cannot:**
    - ‚ùå Change organization settings
    - ‚ùå Manage members
    - ‚ùå Assign roles
    - ‚ùå Delete teams/projects
    
    **Best for:** Regular employees, contributors
  </Tab>
  <Tab value="Viewer">
    **Read-only access**
    
    **Can:**
    - ‚úÖ View organization profile
    - ‚úÖ See public teams/projects
    - ‚úÖ View member list
    - ‚úÖ Follow the organization
    
    **Cannot:**
    - ‚ùå Participate in teams
    - ‚ùå Create projects
    - ‚ùå Change anything
    - ‚ùå Access private content
    
    **Best for:** Contractors, observers, auditors
  </Tab>
</Tabs>

## Best Practices

### For Organization Owners

<Callout type="tip" title="Success Strategies">
  **1. Clear Structure:** Define roles and teams clearly  
  **2. Regular Updates:** Keep profile and info current  
  **3. Active Management:** Review members and permissions regularly  
  **4. Communication:** Use announcements for important updates  
  **5. Security:** Enable 2FA requirements for members  
  **6. Documentation:** Maintain internal docs and guidelines
</Callout>

### Organization Naming

**Good names:**
- ‚úÖ Clear and memorable
- ‚úÖ Represents your purpose
- ‚úÖ Professional
- ‚úÖ Unique

**Avoid:**
- ‚ùå Generic names (e.g., "Team")
- ‚ùå Special characters
- ‚ùå Too long or complex
- ‚ùå Confusing abbreviations

### Member Onboarding

Create a smooth onboarding experience:

1. **Welcome Message:** Send personalized welcome
2. **Documentation:** Share guidelines and resources
3. **Initial Team:** Assign to appropriate team
4. **Buddy System:** Pair with experienced member
5. **First Tasks:** Give clear initial assignments

## Troubleshooting

<Accordions>
  <Accordion title="Can't create organization">
    **Possible reasons:**
    - Account not verified
    - Name already taken
    - Slug not available
    - Need more credits (if applicable)
    
    **Solutions:**
    - Verify your email address
    - Try a different name/slug
    - Check credit balance
    - Contact support
  </Accordion>
  
  <Accordion title="Invite codes not working">
    **Check:**
    - Code hasn't expired
    - Code hasn't been used (if single-use)
    - Recipient email matches (if specified)
    - Organization still exists
    - Recipient has account
  </Accordion>
  
  <Accordion title="Can't remove member">
    **Reasons:**
    - You don't have permission
    - Member is organization owner
    - Member has higher role than you
    
    **Solution:**
    - Ask an admin/owner
    - Transfer ownership first (if owner)
  </Accordion>
  
  <Accordion title="Organization not showing in search">
    **Check:**
    - Organization is set to Public
    - Profile is complete
    - Wait 24 hours for indexing
    - Verify there are no typos in name
  </Accordion>
</Accordions>
