## Creating a Project

### Planning Your Project

Before creating, consider:

| Aspect | Questions to Ask |
|--------|------------------|
| **Purpose** | What will this project deliver? |
| **Scope** | What's included/excluded? |
| **Team** | Who needs to be involved? |
| **Visibility** | Who should see this? |
| **Duration** | Is this temporary or ongoing? |
| **Resources** | What do we need? |

### Creation Process

<Steps>

### Initiate Creation

Team Page ‚Üí **Projects** ‚Üí **Create Project**

### Basic Information

**Project Name**
- Clear and descriptive
- Indicates purpose
- Easy to remember

**Examples:**
- "Website Redesign 2024"
- "Mobile App v2.0"
- "Q4 Marketing Campaign"
- "API Integration Project"

**Slug**
- URL-friendly identifier
- Auto-generated from name
- Customizable
- Permanent (changes carefully!)

### Write Description

Explain the project:

**Include:**
- Project goals and objectives
- What will be delivered
- Timeline (if applicable)
- Key requirements
- Success criteria

**Example:**
```
Redesign the company website with modern UI/UX, improved 
performance, and mobile responsiveness. Launch target: Q2 2024.

Key goals:
- 50% faster load times
- Mobile-first design
- Accessibility compliance (WCAG 2.1)
- SEO optimization
```

### Set Visibility

Choose project visibility:

- ‚≠ï **Public** - Maximum transparency
- ‚≠ï **Internal** - Team members (recommended)
- ‚≠ï **Private** - Members only

### Configure Access

**Join Settings:**
- ‚òê Enable open join (team members can join freely)
- ‚òê Require approval for join requests

**Default Role:**
- Select default role for new members

### Add Project Details

**Optional but recommended:**

**Project Profile:**
- Logo/icon
- Banner image
- Website/repository link
- Documentation link

**Metadata:**
- Project status (Planning, Active, Complete)
- Timeline/milestones
- Priority level
- Tags/labels

### Assign Initial Members

Add your first team:
1. Search team members
2. Assign project roles
3. Set permissions
4. Send notifications

### Create Project Roles

Define project-specific roles:

| Role | Level | Typical Use |
|------|-------|-------------|
| **Project Lead** | Highest | Project manager |
| **Contributor** | Standard | Active workers |
| **Reviewer** | Review | QA, approval |
| **Viewer** | Read-only | Stakeholders |

### Launch Project

Click **Create Project** - you're ready to work! üöÄ

</Steps>

## Project Best Practices

### Organization

<Callout type="tip" title="Project Organization Tips">
  **1. Clear Goals:** Define what success looks like  
  **2. Right Size:** Not too big, not too small  
  **3. Active Ownership:** Assign clear project leads  
  **4. Regular Updates:** Keep stakeholders informed  
  **5. Member Engagement:** Ensure everyone contributes  
  **6. Documentation:** Maintain project docs
</Callout>

### Naming Conventions

**Good project names:**
- ‚úÖ Descriptive and specific
- ‚úÖ Include version/phase if applicable
- ‚úÖ Indicate purpose clearly
- ‚úÖ Professional and clear

**Examples:**
- ‚úÖ "Website Redesign Q1 2024"
- ‚úÖ "Mobile App v2.0 Development"
- ‚úÖ "Customer Portal Integration"
- ‚úÖ "Marketing Campaign - Summer"

**Avoid:**
- ‚ùå Generic names ("Project 1")
- ‚ùå Only dates ("2024-01")
- ‚ùå Internal codes ("PROJ-ABC")
- ‚ùå Unclear abbreviations

## Troubleshooting

<Accordions>
  <Accordion title="Can't create project">
    **Check:**
    - You're in a team
    - You have permission to create projects
    - Team has space for more projects
    - Project name not taken in team
    
    **Solutions:**
    - Ask team admin for permission
    - Try different name
    - Check team limits
    - Verify team membership
  </Accordion>
  
  <Accordion title="Can't add members">
    **Reasons:**
    - Insufficient permissions
    - Person not in team
    - Already a member
    - Project limit reached
    
    **Solutions:**
    - Check your role permissions
    - Ensure person is team member
    - Verify they're not already added
    - Contact project owner
  </Accordion>
  
  <Accordion title="Can't change visibility">
    **Reasons:**
    - Insufficient permissions
    - Team/org policy restrictions
    - Active content prevents change
    
    **Solutions:**
    - Ask project owner
    - Check team settings
    - Review content visibility first
  </Accordion>
  
  <Accordion title="Lost access to project">
    **Possible causes:**
    - Removed from project
    - Removed from team
    - Role changed to lower permission
    - Project visibility changed
    
    **Solutions:**
    - Check with project admin
    - Verify team membership
    - Request re-addition
    - Check email for notifications
  </Accordion>
</Accordions>
